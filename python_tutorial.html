版本命名方式

    $ python -V # check python version
    Python V.x.y # Ex. 3.3.0 or 2.7.3

  • V, x 都是大改版，像function, library的調整。可以查官網的change log
  • 相同 V 的 code 都可以向後相容，所以理論上2.5的code在2.7上也可以執行
  • y 是 bug fix 跟提昇效能
  • V = 2 or 3，是相容性的例外（後述）

Python 3 (Py3k)

  • 把一些舊時代的垃圾清掉，整合 library。
    Ex. urllib, urllib2; popen2(), popen3(), popen4()
  • Syntax 上的調整。
    Ex. print -> print(), with ... as, list comprehension
  • string 原生對 unicode 支援
  • 2.5 = 3.0, 2.6 = 3.1, 2.7 = 3.2
  • 3.3(2012.09)為全新版本，2.X不再更新功能
  • 建議直接學 3.3，有問題再回到 2.7

相對的缺點

  • 有些套件還沒相容（停止開發、大套件），但這是趨勢且大部份都已經相容。
    例外：Scipy ( 3.3 還再等 )
  • 效能差於 2.7.3

語言介紹

  • dynamic typing, intepretion, scipt, OOP
  • 所以可以很像 Matlab，可以逐行執行/運行腳本
  • 變數不用宣告，型別不固定
  • （不過是強型別，不會自動型別轉換）
  • 但他也可以使用 OOP 的方式來寫（萬物皆 object）
